FAQ (quick):

  • Process vs Thread: A process has its own memory space; threads share process memory.
  • Context switch: CPU switching from one process/thread to another.
  • Deadlock: Circular waiting for resources, none can proceed.

A process is an instance of a running program that has its own memory space, program counter, and resources managed by the OS.

A thread is the smallest unit of CPU execution within a process. Threads in the same process share memory and resources, allowing lightweight concurrency.

Context switching saves the CPU state of one process/thread and loads another‚Äôs state. It's expensive because it involves memory management, cache/TLB flushes, and CPU register operations.

Preemptive scheduling can interrupt a running process to start another (e.g., Round Robin). Non-preemptive scheduling runs a process to completion or until it blocks (e.g., FCFS).

A critical section accesses shared resources. Protect with synchronization primitives: mutexes, semaphores, locks, or monitor constructs to ensure mutual exclusion.

Deadlock: processes wait forever for resources. Necessary conditions: mutual exclusion, hold and wait, no preemption, circular wait.

Prevent by removing one of the necessary conditions, use resource ordering, allow preemption, or use deadlock detection with recovery (rollback, kill process).

Paging divides memory into fixed-size pages and maps them to physical frames to avoid external fragmentation and provide efficient memory allocation.

Virtual memory gives processes the illusion of a large contiguous address space by mapping virtual addresses to physical memory and using disk space (swap) as extension.

Thrashing occurs when the OS spends more time swapping pages in/out than executing processes due to insufficient physical memory and excessive page faults.

FIFO removes oldest page; LRU removes least recently used page. LRU approximates optimal behavior when recent use predicts future use.

Swapping moves entire processes between memory and disk to free memory; it's heavier than paging and used in older OS designs or under memory pressure.

System calls are kernel-provided APIs allowing user programs to request privileged operations (I/O, process control, memory management) from the OS.

IPC allows processes to exchange data using pipes, sockets, shared memory, message queues, and signals, each with trade-offs in speed and complexity.

Mutex is a binary lock for mutual exclusion; semaphore is a counter allowing limited concurrent access. Mutex associates with a single owner; semaphores do not.

Starvation occurs when a process never gets resources because others are favored. Avoid by using fair scheduling, aging, or priority adjustments.

Scheduler uses policy (priority, fairness) and algorithm (RR, SJF, Multilevel Queue) considering CPU burst, waiting time, and priority to pick processes.

Kernel mode has full hardware access for privileged operations; user mode restricts access so user programs cannot corrupt system state directly.

I/O scheduling orders disk requests to improve throughput and reduce seek time (algorithms: FCFS, SSTF, SCAN, C-SCAN).

OSes use paging to eliminate external fragmentation; compaction and dynamic allocation strategies reduce internal fragmentation; virtual memory isolates processes.

FAQ (quick):

  • OSI vs TCP/IP: OSI is conceptual 7-layer model; TCP/IP is practical 4-layer model.
  • IP vs TCP vs UDP: IP routes packets; TCP is reliable stream; UDP is connectionless.
  • What is routing? Determining best path to send packets across networks.

TCP is connection-oriented and reliable with flow/congestion control; UDP is connectionless, lightweight, and suitable for real-time apps where occasional loss is acceptable.

Seven layers: Physical, Data Link, Network, Transport, Session, Presentation, Application ‚Äî each handles specific functions from bits to application services.

Subnet divides a network into smaller parts. Subnet mask (e.g., 255.255.255.0) separates network and host bits in an IP address.

NAT maps private IPs to a public IP, conserving IPv4 addresses and allowing multiple hosts to share one public address while hiding internal topology.

DNS translates domain names to IP addresses using hierarchical servers (root, TLD, authoritative). Clients query resolvers which follow referrals to get records.

Switching operates within a LAN (frame forwarding using MACs). Routing operates between networks (packet forwarding using IPs and routing tables).

Client sends SYN, server replies SYN-ACK, client sends ACK ‚Äî connection established. This ensures both sides agree on initial sequence numbers.

Flow control prevents sender from overwhelming receiver (e.g., TCP window). Congestion control prevents network overload (e.g., TCP Reno/CUBIC).

ARP (Address Resolution Protocol) maps IPv4 addresses to MAC addresses on a local network segment so frames can be delivered at Data Link layer.

Sockets are endpoints for network communication (IP + port). They provide a programming interface for TCP/UDP communication.

MTU is the maximum transmission unit (largest packet a link can carry). If packet > MTU, IP fragmentation splits packet into smaller fragments.

HTTPS = HTTP over TLS. TLS provides encryption, integrity, and server (and optionally client) authentication using certificates and keys.

Distance-vector (e.g., RIP) and link-state (e.g., OSPF). Distance-vector uses periodic updates; link-state floods link info and computes shortest path (Dijkstra).

Quality of Service prioritizes certain traffic to meet latency, jitter, and throughput requirements (useful for VoIP, streaming).

TCP uses sequence numbers, ACKs, retransmissions on timeouts, checksums, and congestion/flow control to provide reliable, ordered delivery.

VLAN logically segments a switch into multiple broadcast domains, improving security and network organization without separate physical switches.

Symmetric uses same key for encrypt/decrypt (fast); asymmetric uses public/private key pairs (slower, used for key exchange & signatures).

Port is endpoint identifier within IP. Well-known ports: 80 (HTTP), 443 (HTTPS), 22 (SSH), 25 (SMTP), 53 (DNS).

UDP doesn't guarantee delivery ‚Äî apps implement reliability if needed (e.g., RTP with RTCP or custom ACKs) for low-latency use-cases.

Link-state has faster convergence and avoids routing loops by using global topology knowledge and Dijkstra; distance-vector is simpler but slower and can loop.

FAQ (quick):

  • ACID vs BASE: ACID ensures strong consistency; BASE prefers availability and eventual consistency.
  • Normalization removes redundancy (1NF, 2NF, 3NF).
  • Index speeds queries but adds write overhead.

Normalization organizes tables to reduce redundancy and improve integrity by applying normal forms (1NF, 2NF, 3NF, BCNF).

ACID: Atomicity (all or nothing), Consistency (DB rules preserved), Isolation (transactions don't interfere), Durability (committed changes persist).

Index speeds lookups. Types: B-tree (balanced), Hash (exact-match), Bitmap (low-cardinality), and clustered vs non-clustered indexes.

A transaction is a sequence of DB operations that execute as a single unit, maintaining ACID properties; it‚Äôs committed or rolled back.

Isolation levels: Read Uncommitted, Read Committed, Repeatable Read, Serializable ‚Äî trade-offs between consistency and concurrency (phenomena: dirty read, non-repeatable read, phantom read).

Denormalization intentionally adds redundancy to improve read performance or simplify queries in high-read systems, at cost of update complexity.

Primary key uniquely identifies a row. Foreign key references primary key of another table to enforce referential integrity.

A view is a virtual table defined by a query; it simplifies complex queries, encapsulates logic, and can provide security abstraction.

Indexes speed reads but add storage and slow writes (inserts/updates) because indexes must be maintained; choose indexes strategically.

Partitioning splits large tables into smaller pieces (horizontal/vertical) to improve manageability and query performance across partitions.

Transaction log records DB changes for durability and recovery; used to replay or roll back transactions during crash recovery.

Concurrency control ensures correct results when multiple transactions run concurrently via locking protocols, MVCC, and optimistic concurrency control.

Multi-Version Concurrency Control keeps multiple versions of data to allow readers to see a consistent snapshot while writers create new versions, improving concurrency.

DB deadlock: transactions wait for locks held by each other. Handled by detection (wait-for graph) and resolution (abort/rollback one transaction).

Schema is the database structure (design). Instance is the actual data stored at a point in time.

Use NoSQL for flexible schema, horizontal scaling, and high write throughput. Use SQL for complex transactions, strong consistency, and relational data.

Eventual consistency means updates propagate asynchronously; replicas become consistent over time, trading immediate consistency for availability and partition tolerance.

Join combines rows from tables: INNER (intersection), LEFT (all left + matched right), RIGHT, FULL OUTER (all), CROSS (Cartesian product).

Use EXPLAIN, add/select proper indexes, avoid SELECT *, optimize joins, use caching, and consider schema/partition/denormalization for heavy reads.

Replication duplicates data across nodes. Types: master-slave (primary-secondary), master-master (multi-primary), synchronous vs asynchronous replication for consistency trade-offs.

FAQ (quick):

  • OOP pillars: Encapsulation, Abstraction, Inheritance, Polymorphism.
  • Difference: Java has garbage collection; C++ provides manual memory control and multiple inheritance (via virtual inheritance).

Encapsulation hides internal state and exposes behavior via methods, bundling data and methods together to protect object integrity.

Inheritance allows a class to derive from another. Types: single, multiple (C++), multilevel, hierarchical. Java supports single class inheritance (interfaces for multiple behavior).

Polymorphism lets objects take multiple forms: compile-time (method overloading) and runtime (method overriding with virtual/override keywords or dynamic dispatch).

Abstraction exposes essential features while hiding implementation details, using abstract classes or interfaces to define contracts.

Java uses garbage collection for automatic memory management; C++ uses manual allocation/deallocation (new/delete) and RAII patterns for deterministic cleanup.

Overloading: same method name, different params (compile-time). Overriding: subclass provides different implementation for base method (runtime polymorphism).

Virtual function enables dynamic dispatch: calls resolved at runtime based on actual object's type allowing polymorphic behavior in C++.

Interface defines contract (methods without state) and allows multiple inheritance of type; abstract class can have state and partial implementation. Java interfaces can have default methods now.

RAII (Resource Acquisition Is Initialization) ties resource lifetime to object lifetime so resources are released in destructors, ensuring exception-safe cleanup.

Diamond problem arises with multiple inheritance; solve using virtual inheritance to ensure only one base subobject is present.

Constructor initializes an object. Destructor (C++) cleans up before object destruction. Java has finalize (deprecated) and relies on GC for cleanup.

Both have try-catch. Java enforces checked exceptions while C++ does not. C++ destructors help cleanup during stack unwinding; Java uses finally blocks for cleanup.

Composition (has-a) embeds objects as members; prefer composition for flexibility. Inheritance (is-a) is used for specialization but can lead to tight coupling.

SOLID: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion ‚Äî principles for maintainable OOP design.

Subtypes must be substitutable for their base types without altering program correctness ‚Äî derived classes should honor base contracts.

Dependency injection provides dependencies from outside a class (constructor/setter) to reduce coupling and improve testability.

Final methods cannot be overridden. Static methods are bound at compile time and cannot be overridden; they can be hidden by subclass methods with same signature.

Smart pointers (unique_ptr, shared_ptr, weak_ptr) automate memory management with ownership semantics to avoid leaks and dangling pointers.

Use interface to define behavior without state and allow multiple type inheritance; use abstract class to share code and state among related classes.

Object slicing occurs when a derived class object is assigned to a base class object by value, losing derived-specific data; use pointers/references to avoid slicing.

Quick list: JVM, JIT, garbage collection, memory model, pointers (C++), references, templates, STL, exceptions, threading.

JRE provides runtime environment (libraries + JVM); JVM is the virtual machine that executes Java bytecode and provides memory management and JIT compilation.

JIT (Just-In-Time) compiles frequently executed bytecode into native machine code at runtime for performance improvements.

Java memory model covers how threads interact through memory, visibility, ordering, and synchronization primitives to avoid race conditions.

Garbage collectors: Serial, Parallel, CMS, G1, ZGC ‚Äî algorithms vary in throughput and pause-time trade-offs; GC reclaims unreachable objects automatically.

Pointer stores memory address and can be null/change; reference is an alias and must be initialized, cannot be reseated or null (safer syntax).

Templates enable generic programming; function and class templates are instantiated by compiler for types used, providing type-safe code reuse.

STL (Standard Template Library) has containers (vector, list, map), iterators, algorithms (sort, find), and functors to write generic, efficient code.

Exception safety levels: basic (no leaks), strong (transactional rollback), and no-throw (guarantee). Use RAII and smart pointers to ensure safety.

List allows ordered duplicates; Set stores unique elements; Map stores key-value pairs with unique keys. Implementations differ: ArrayList vs LinkedList, HashSet vs TreeSet, HashMap vs TreeMap.

Synchronization prevents concurrent access issues. Use synchronized keyword, locks, or concurrent utilities (ConcurrentHashMap, ReentrantLock) to control thread access.

volatile ensures reads/writes go to main memory and enforces visibility; it doesn't provide atomicity for compound actions.

Move semantics (rvalue references) allow resources to be moved instead of copied, improving performance by transferring ownership without expensive allocations.

Generics provide compile-time type safety by allowing classes and methods to operate on specified types without casting (e.g., List ).

Undefined behavior is behavior not defined by the C++ standard (e.g., dereferencing null). It can lead to unpredictable results and must be avoided.

Use smart pointers (unique_ptr/shared_ptr), RAII, and ensure every new has a matching delete or use container-managed resources to avoid leaks.

finalize is deprecated and unpredictable; try-with-resources ensures deterministic resource cleanup by auto-closing Closeable resources at block exit.

Template specialization customizes behavior for specific types so you can provide a different implementation for particular template parameters.

Classloader loads classes into JVM. Types: bootstrap, extension, and application classloaders. Custom classloaders can load classes from nonstandard sources.

Throwable -> Error (serious problems) and Exception -> RuntimeException (unchecked) and checked exceptions. Handle checked exceptions or declare them.

SFINAE (Substitution Failure Is Not An Error) is a template metaprogramming technique where invalid substitutions remove overloads, enabling compile-time selection. Useful in template-based dispatch.

FAQ (quick):

  • Focus: scalability, availability, trade-offs, CAP theorem, caching, data partitioning.
  • Start with requirements, constraints, API design, high-level components, data models, and bottlenecks.

Clarify requirements, scope, and constraints. Estimate traffic, define APIs, sketch high-level components, then drill into storage, caching, and bottlenecks.

Load balancers distribute requests across servers. Types: Layer-4 (IP/TCP) and Layer-7 (HTTP-aware). Algorithms: round-robin, least connections, weighted.

Caching stores frequently accessed data close to consumers (in-memory caches like Redis, CDN edge caches) to reduce latency and load on origin systems.

Sharding splits data horizontally across multiple DB instances for scale. Choose shard key carefully to balance load and minimize cross-shard queries.

CAP: Consistency, Availability, Partition tolerance ‚Äî in the presence of network partitions you can only guarantee two of the three. Real systems choose trade-offs per requirements.

Define API, use base62 short codes, store mapping in DB, use caching for hot keys, handle collisions, consider expiry, and scale DB via sharding or consistent hashing.

Consistent hashing maps keys to a circular space of nodes; when nodes change, only a subset of keys remap‚Äîuseful for distributed caches and partitioned stores.

Eventual consistency means updates propagate asynchronously and replicas converge over time; it's chosen for availability and partition tolerance trade-offs.

Use websockets for real-time delivery, stateless backend, message store in DB, deliver via fanout services or pub/sub, scale using shards and sticky sessions for stateful connections.

Message queues (Kafka, RabbitMQ) decouple producers and consumers, buffer bursts, enable asynchronous processing, and improve system resilience and scalability.

Connection pooling reuses DB connections reducing overhead of establishing connections, improving throughput and latency for DB-heavy workloads.

Backpressure signals producers to slow down when consumers can't keep up, preventing overload and memory/exhaustion issues in pipelines and streaming systems.

Write-through writes to cache and DB synchronously (safer, slower). Write-back writes to cache and lazily to DB (faster, riskier if cache lost).

CQRS (Command Query Responsibility Segregation) separates read and write models allowing optimized data views and scaling reads/writes independently.

Use redundancy, stateless services, health checks, failover, multi-AZ deployments, data replication, and automated recovery to minimize downtime.

Bloom filter is a space-efficient probabilistic data structure to test set membership with false positives allowed; used in caching and deduplication checks.

Estimate requests per second (QPS) from user base and patterns, estimate resource per request (CPU, memory), multiply and add headroom; size servers and caches accordingly.

Rate limiting caps requests per client/time window. Throttling slows or rejects excessive requests to protect system stability often using token bucket or leaky bucket algorithms.

Common bottlenecks: DB, network, CPU, disk I/O. Identify using metrics, APM traces, profiling, slow query logs, and load testing to pinpoint where latency originates.

Use versioning, timestamps, conflict resolution strategies (last-write-wins, CRDTs, application-level merges), and background reconciliation jobs to converge replicas.
